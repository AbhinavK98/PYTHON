# DSA Foundations Repository

Welcome. This is **not** a LeetCode problem bank.

This repository teaches the **thinking process** you need *before* solving interview problems.
If you master these foundations, patterns on LeetCode become much easier to recognize and explain.

---

## What is this?

A permanent revision guide covering:

- How to think in DSA
- Time and space complexity
- Recursion (deep dive)
- Searching
- Sorting
- Bit manipulation
- Mathematics for interviews
- Interview templates

Every topic answers: **WHY**, **HOW**, **WHEN**, **WHAT**.

---

## How to Study Each Topic

1. Read the topic `README.md` (overview + analogies).
2. Read theory files in numbered order.
3. Draw ASCII diagrams on paper while reading.
4. Do dry runs manually.
5. Revise with cheat sheets (5 min / 1 min versions).
6. Attempt practice questions without looking at answers.

---

## Progress Checklist

- [ ] I can explain any algorithm in plain English.
- [ ] I can derive Big O without memorizing formulas.
- [ ] I can trace recursion on paper (call stack + returns).
- [ ] I can implement binary search without off-by-one bugs.
- [ ] I can compare sorting algorithms and pick the right one.
- [ ] I can recognize when bit tricks apply.
- [ ] I have memorized core interview templates.

---

## Interview Readiness (Foundations Level)

- [ ] Given code, I can state time and space complexity.
- [ ] I can discuss trade-offs (time vs space, iterative vs recursive).
- [ ] I can write brute force first, then optimize with reasoning.
- [ ] I can explain *why* an approach works, not just *what* it does.
