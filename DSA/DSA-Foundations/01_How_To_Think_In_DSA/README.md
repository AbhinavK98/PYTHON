# How To Think In DSA

## What is it?

A structured way to approach any coding problem: understand, plan, implement, verify.

## Why do we need it?

Random coding fails in interviews. Interviewers reward clear thinking under pressure.

## Real-world analogy

Like cooking: read recipe (problem), gather ingredients (constraints), prep (brute force), cook (optimal), taste (test).

## Visualization

```
Problem -> Constraints -> Examples -> Brute Force -> Optimize -> Code -> Test -> Explain
```

## Where interviewers ask it

Every interview round. Especially when candidate jumps to code too fast.

## Advantages

- Reduces panic
- Shows maturity
- Catches edge cases early

## Disadvantages

- Takes 2-3 minutes upfront (worth it)

## When NOT to use it

Never skip in interviews. In contests with known patterns, you may shorten.

## Interview Tricks

- Say your plan out loud
- Write examples first
- Name the pattern before coding
